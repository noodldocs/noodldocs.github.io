Key elements of a list

Key elements of a list

A list is build out of three ingredients:

1. A dataset with data to populate our list with items

2. A visual component that controls how the list item.

3. A repeater node that creates one list item component per data record

Let's start by looking at the data

 

Static data

There is many ways that we can get data sets into our project. A very easy way is to use the Static Array node.

This node acts like a text file and supports two very common data formats being JSON and CSV. We have already inserted a small dataset following the JSON format.

This dataset contains images from Unsplash.com along with the names of the artists behind the images.

 

Components

Components is a great way to break your project into smaller building blocks. A component in Noodl is a set of nodes that is represented as a new node.

A main benefit of using components is that they become reusable and you can place multiple instances of that component in a project.

In the sidebar you can see all the components in the project, and from here you can also create new components.

Noodl have three components types being Visual, Logic, and Page Components. In this lesson we will be using Visual Components

 

Visual components

As you can see in the project, we already have a few visual nodes that makes up a simple layout of an image and a text node with artist's name.

We will be creating a visual component from this simple layout, so we can repeat that component for each record in our dataset.

Repeater node

With our new List item component we can manually insert as many instances as we want into our layout.

But even better is that we can use our dataset from the Static Array node to generate the List item component for each record in the data.

To do that we want to use the Repeater node. This visual node is placed in the visual hirachy and from the property panel we can set what component the repeater should repeat.

Now we just need to connect the data from the Static Array node to the Repeater's items input.

Connecting data

As you might notice, we got 5 images now (one for each image + author pair in the dataset.)

However, right now the repeater node is just repeating our List item component 5 times.

Our next step is to connect the data values in the dataset to the visual nodes in our List item component. And to do that we will be using an Object node.

 

The Object node

The Object node lets us access data from objects. The Static array node automatically creates data objects, each with an unique id, for each record in the data set

When we use an Object node inside of a component generated by a Repeater node, the Repeater can provide us with the Id for each of the data object. This can be set from the property panel.

Once we have set the Object node to get its id from the Repeater, we can hover over the object node and it will show it's content.

Object properties

When we hover over the object we can see the data properties available. We can make these properties available as outputs on the Object node by clicking the plus icon in the Object's property panel

Type in the property name so it's identical to what we have in the dataset. Remember that the names needs to be identical.

Now the outputs are available as outputs and we can connect them to other nodes in the component.
