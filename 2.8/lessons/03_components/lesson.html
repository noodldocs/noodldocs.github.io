Components

Components

Components are used to break your project into smaller building blocks. There is two reasons why you want this:

1) It's a good way to break your project into multiple building blocks so we don't end up with hugh node graphs.

2) Components are reusable so we can place multiple instances of that component in a project. This becomes extra important when we ant to make our app data-driven.

In the sidebar, you get an overview of all the components in the project, and from here you can also create new components.

Noodl has three components types: Visual, Logic, and Page Components. We will be using visual components in this lesson

 

Visual components

Visual components

We can turn a set of visual nodes (like our Card Item) into a Visual Component to create multiple instances of the component. In our app we want multiple Card Items, so thats a perfect case to turn something into a component.

Creating a visual component is done by clicking the "+" icon in the component section of the sidebar. Let's do that in the next step.

Component Inputs

Component Inputs

When we changed the bottom margin on the Card Item component, the change applied to all instances. However, some properties we want overwrite be unique for each instance, like the displayed text, the image and the background color.

We can achieve this by using the Component Input Node. Through that we can create "ports" and connect those ports to the input properties of our visual nodes that we want to overwrite.

Once we do that, the Card Item component node will get Input Properties that we can set to overwrite the default values.

Overwriting Component inputs

Overwriting Component inputs

When we have connected ports from our Component Input node to input properties of our visual nodes, these properties are now exposed on the component node wherever we are using it.

This means that we can style each instance of the component to our liking. And we can always connect up more properties, if we want to control more properties.
