User created data

Welcome back! In this sixth lesson we will learn how to capture data from a form of UI Inputs and save that data using Objects and Arrays, to display the data in another place.

In this lesson we will make a new page with a form for creating new task cards that we will display on the Task page.

 

Creating forms

Creating forms

Noodl comes with multiple ways of creating forms for capturing user data. The simplest way is to simple use Noodl core nodes for adding the desired UI Controls for capturing user inputs.

Buttons and Text inputs are some we already used, but we can also add dropdowns, sliders, checkboxes, date pickers and much more with a few clicks.

We will be build out a simple form page that enables the user to create their own task cards. But first lets create a new page for the form.

The Dropdown node

Dropdown node

With the Dropdown node we can quickly add a way for users to select a category for their task.

The dropdown node needs an array of options where each option has a label, being what shown in the UI. and a value, which is the returned value when an option is selected.

Datepicker and other Prefabs

Datepicker and other Prefabs

Besides the core nodes, Noodl also comes with a bunch of pre-made components called Prefabs that can be found in the Node Picker.

Prefabs are often more advanced components that still can be customized as needed, as most of them are built with regular Noodl nodes.

When you clone a prefab it gets added to your component list and can be dragged in to your node graph

Store Objects in Arrays

Store Objects in Arrays

To actual create an object, we need to trigger the "Do" action on the Create New Object node.

Ones the "Do" action is triggered, a New object is created with whatever data is given through the UI controls, and a unique Object Id is outputted which we can use anywhere in our app to access the information.

But we need to store that Id, and that we can do with an Array and the Insert Object into Array node.

Using the Tasks Array

Using the Tasks Array

Before we can see our newly created cards on the Task Page, we need to swap out the Static array node with an Array node with the same id ("Tasks") as the Insert Object Into Array node.

Because Array ID's are global we can access the data of an array everywhere we have an Array node with the same Id.

A quick note on Arrays and Objects is that they only hold data locally and are reset when the page or project is refreshed. When this lesson is done we will show how to make the data persist.

But for now let go and swap out the Static Array node with a Array node.
