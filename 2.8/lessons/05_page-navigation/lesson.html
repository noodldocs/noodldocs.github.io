Key elements of a navigation system

Key elements of a navigation system

All Noodl projects starts with a simple navigation system in place. We have already learned how to navigate between existing pages, but lets take step back a look at the three key elements for working with navigation in Noodl:

  1. Multiple Page Components that each contains a page of an application. We already have a Task Page and a Start Page.

  2. A Page Router node, that works works as a container that controls what Page component to show. This node is often placed in the App component.

  3. A Navigate node that lets you navigate from one page to another.

 

Page Router node

Page Router node

The Page Router node is a visual node that acts as a container for pages. Any page you navigate to will be rendered inside the Page Router and fill out the space available.

Navigation menu and footer often live next to the Page Router in the Node Graph so they don't need to be inserted in every page.

If we select the Page Router we can see all the Pages that are part of it, and we can also set which page we want to act as the start page.

The Page component

Repeater node

Repeater node

The Repeater node is used to generate visual components based on objects in an array. It takes a visual component as a template that then will be rendered where the Repeater node is placed in the Dom tree.

The repeater needs an array to generate from, and for each object in the array, a component instance is generated and rendered.

Making clickable cards

Making clickable cards

Now we have built the basic system for showing cards (or any other type of list) from data. But to take this a step further we can also implement a way to make each card clickable and get its associated object so we can use that data somewhere else.

To do this we first need to add a Component Output node into a our Card component, so we can get data out of the card as well.

 

Component Output node

Component Output node

Just as we can use a Component Input node to get data and signals into a component, we can use a Component Output node to get data and signals out of a component.

In our case we want to make the Card component clickable, so we can receive a signal from the card when its clicked.

Repeaters and Component Outputs

Repeaters and Component Outputs

If we just had a single Card component we would now be able to get the click event in the node graph where the Card component is placed.

But as we are using a Repeater node to generate four Card components, we need a way to know which of the four we click.

Earlier in this lesson we talked about how each object in the array has an unique Id. This now becomes handy as we can make the Repeater node give us the object id of the card instance we click.

 

Object node

Object node

The Object node gives us a way to access an object and its data anywhere in a project if we give it the unique id of an object.

Objects are in general a good way to store data in one node graph and access it again in another node graph.

We can use the Object node in combination with the Repeater node to receive the object data from a clicked Card.
